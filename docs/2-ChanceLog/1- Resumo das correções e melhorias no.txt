Resumo das correções e melhorias no UserService

1. Atomicidade na criação do usuário
   Foi adicionado `transaction.atomic()` no método `create_user` para garantir que a criação do `User` e do `UserProfile` aconteçam na mesma transação.
   Assim, se ocorrer um erro na criação do perfil, o usuário também é revertido, evitando inconsistência no banco.

2. Correção da tipagem com Optional
   Os parâmetros opcionais foram corrigidos para usar a sintaxe correta de tipagem do Python:

Errado:
avatar=Optional[str] = None

Correto:
avatar: Optional[str] = None

Isso também foi aplicado para `bio` e `daily_goal_minutes`.

3. Correção de erro de ordem de parâmetros
   O parâmetro `daily_goal_minutes` estava sem valor padrão após parâmetros opcionais, o que gera erro de sintaxe em Python.
   Foi corrigido para:

daily_goal_minutes: Optional[int] = None

4. Correção de erro de sintaxe (`Not`)
   Foi corrigido:

if bio is Not None

para

if bio is not None

pois Python é case-sensitive.

5. Correção de bug lógico no save()
   O `profile.save()` estava dentro do bloco `if daily_goal_minutes`, o que faria o perfil não ser salvo caso apenas `avatar` ou `bio` fossem alterados.
   O `save()` foi movido para fora do bloco condicional.

6. Correção do return
   O `return profile` estava dentro do bloco condicional e poderia resultar em retorno `None`.
   Agora ele está fora do bloco, garantindo que o método sempre retorne o perfil.

7. Pequenas correções de documentação
   Foram corrigidos erros de digitação nas docstrings, como:

* "osquestra" → "orquestra"
* "relacionada" → "relacionadas"

Resultado final:
O serviço agora está mais seguro, tipado corretamente, sem erros de sintaxe e com lógica de persistência correta.
